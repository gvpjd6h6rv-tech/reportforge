ReportForge v19.8 — Stale minHeight Fix (Real Root Cause)
==========================================================
Date: 2026-03-19

BUG
---
White area appears below sections after resize (expand then shrink).

REAL ROOT CAUSE
---------------
SectionResizeEngine.onMouseMove updated canvas-layer style.height but
NOT style.minHeight during live drag.

Timeline of the bug:
  1. Page loads: height=531px, minHeight=531px (from CanvasLayoutEngine boot)
  2. User drags section to expand: height=666px, minHeight=531px (stale!)
  3. User drags section to shrink: height=441px, minHeight=531px ← CONFLICT
  4. Browser resolves: offsetHeight = max(441, 531) = 531px
  5. White area = 531 - 441 = 90px gap below last section

The same bug existed in CanvasLayoutEngine.js and CanvasEngineV19.js
which set minHeight independently from height, allowing them to diverge.

FIX
---
Rule: minHeight must ALWAYS equal height. Never set independently.
Implementation: everywhere height is written, minHeight is written to the same value.

designer/crystal-reports-designer-v4.html — SectionResizeEngine.onMouseMove:
  BEFORE:
    cl.style.height = RF.Geometry.scale(DS.getTotalHeight()) + 'px';
  AFTER:
    const clH = RF.Geometry.scale(DS.getTotalHeight()) + 'px';
    cl.style.height    = clH;
    cl.style.minHeight = clH;   // v19.8: always in sync

engines/CanvasLayoutEngine.js — _apply():
  BEFORE:
    cl.style.width     = `${scaledW}px`;
    cl.style.minHeight = `${scaledH}px`;
  AFTER:
    cl.style.width     = `${scaledW}px`;
    cl.style.height = cl.style.minHeight = `${scaledH}px`;

engines/CanvasEngineV19.js — same change as CanvasLayoutEngine.js.

FILES CHANGED
-------------
  designer/crystal-reports-designer-v4.html   (SectionResizeEngine.onMouseMove)
  engines/CanvasLayoutEngine.js               (_apply: minHeight synced to height)
  engines/CanvasEngineV19.js                  (_apply: minHeight synced to height)
