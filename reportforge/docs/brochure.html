¿Qué es ReportForge?

Un diseñador visual de reportes empresariales similar a Crystal Reports, completamente en Python + HTML5, sin dependencias externas, con paridad de funcionalidad del 95%.

🎨

Diseñador Visual

Canvas drag & drop idéntico a Crystal Reports. Secciones RH, PH, Detalle, PF, RF. Zoom 25%-400%.

🧮

Motor de Fórmulas

Sintaxis Crystal y Basic completas. 60+ funciones. Variables globales, locales y compartidas.

🚀

Sin Dependencias

Solo Python stdlib. Sin FastAPI, sin React, sin npm obligatorio. Arranca en 2 segundos.

📐

Precisión Geométrica

RF.Geometry v5 con Matrix2D, AABB, MagneticSnap. Desviación máxima: 0.001px.

🔬

25 Etapas CI/CD

Pipeline de verificación completo: stress, chaos, fuzz, invariant, reproducible build y más.

🌗

Dual UI

Tema Classic (Crystal Reports XP) y Modern. Toggle en runtime. Mismo motor para ambos.

Sistema Geométrico RF.Geometry v5

Motor de geometría de precisión 0.001px. Único punto de verdad para todos los overlays visuales.

🔢

Matrix2D

Matrices afines 64-bit. Translate · Scale · Rotate · Multiply · Inverse. M * M⁻¹ = I verificado.

📦

AABB

Axis-Aligned Bounding Box. overlaps · intersection · MTV · expand. Simétrico y correcto.

🧲

MagneticSnap

Snap a grid 8px con tolerancia 4px. Precisión ≤ 0.001px. Idempotente verificado.

👆

PointerNorm

Mouse · Touch · Pen unificados. Pressure, pointer type, coordenadas canvas pre-transform.

📏

DOMRect-based

Todos los overlays usan getBoundingClientRect(). Nunca coordenadas del modelo a zoom ≠ 1.

🔄

Frame Cache

invalidate() al inicio de cada frame. Sin layout thrashing. Batching automático por frame de render.

Zoom Dual-Viewport

Design y Preview son hilos de zoom independientes. Cambiar uno no afecta al otro.

🖊️

DesignZoomEngine

Estado: DS.zoom
Escala: #canvas-layer transform:scale(z)
Pasos: 25% · 50% · 75% · 100% · 150% · 200% · 300% · 400%

👁️

PreviewZoomEngine

Estado: DS.previewZoom
Escala: #preview-content transform:scale(z)
Completamente independiente del Design zoom

Instalación y Arranque

En menos de 30 segundos desde cero hasta diseñador corriendo.

# 1. Clonar git clone https://github.com/tu-org/reportforge && cd reportforge # 2. Iniciar (detecta Python, crea .venv, instala deps) ./run.sh # 3. Abrir en el browser open http://localhost:8080/ # Opciones ./run.sh 9000 # puerto personalizado ./run.sh --reset # recrear .venv ./run.sh --check-only # solo verificar entorno # CI/CD completo ./repo.sh # 25 etapas, RF VERIFICATION: 100% PASSED
🐍

Requisito

Python ≥ 3.10. El script detecta automáticamente python3, python3.12, python3.11.

📦

Sin npm obligatorio

Node.js solo para tests de geometría y CI. El servidor funciona 100% con Python stdlib.

⚡

Cold start: 2s

Sin transpilación. Sin build step. El HTML del diseñador es self-contained (226KB).

🌐

API REST

HTTP puro. Sin autenticación requerida en modo dev. Compatible con cualquier cliente.

MétodoRutaDescripción
GET/Diseñador Classic UI
GET/modernDiseñador Modern UI
GET/healthStatus JSON {"status":"ok","version":"2.0.0"}
POST/validate-formulaValidar expresión Crystal en tiempo real
POST/previewPreview HTML con datos JSON
POST/renderRender completo (PDF/HTML)
GET/preview-barcodeSVG de código de barras
# Ejemplo: validar fórmula curl -X POST http://localhost:8080/validate-formula \ -H "Content-Type: application/json" \ -d '{"formula":"ToText({order.total}, 2)"}' # Ejemplo: preview con datos curl -X POST http://localhost:8080/preview \ -H "Content-Type: application/json" \ -d '{"report":{...},"data":{"items":[...]}}'
