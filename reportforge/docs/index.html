🧩 Overview

What is ReportForge?

A Python-native enterprise reporting engine that renders pixel-perfect PDF reports from a JSON layout specification. It replaces Crystal Reports and JasperReports with a modern, cloud-native alternative.

🎨

Visual Designer

WYSIWYG drag-and-drop editor. Absolute positioning, snap-to-grid, layer manager, alignment guides. What you see is exactly what renders.

📊

Full Expression Engine

Arithmetic, conditionals, if/then/else, ternary, running totals, aggregations, template variables like {now()} and {uuid()}.

⚡

REST API + Docker

Production FastAPI server with multi-tenant support. Render PDFs via HTTP. Deploy in minutes with Docker Compose.

🔄

JRXML Compatibility

Import existing JasperReports layouts. Parse JRXML → ReportForge layout → PDF. Full round-trip conversion.

🌐

Live Data Sources

Pull report data from REST APIs, JSON files, or inline payloads. Specify dataSource: "https://api/data" directly in the layout.

🏢

SaaS Multi-Tenant

Per-tenant CSS themes, logo injection, default parameters, template registry. Deploy once, serve hundreds of customers.

📡 REST API

REST API Reference

Full OpenAPI documentation available at /docs. All endpoints accept JSON and return PDF, HTML, XLSX, CSV, or PNG.

POST /render Render layout → PDF/HTML/XLSX/CSV 
# curl example
curl -X POST http://localhost:8000/render \
  -H "Content-Type: application/json" \
  -d '{
    "layout": "examples/enterprise_layout.rfd.json",
    "data":   {"items": [...]},
    "params": {"startDate": "2026-01-01"},
    "tenant": "acme",
    "format": "pdf"
  }' \
  --output report.pdf
POST /designer-preview Live preview from layout JSON (designer use) 
curl -X POST http://localhost:8000/designer-preview \
  -H "Content-Type: application/json" \
  -d '{
    "layout": { "name":"Test", "sections":[...], "elements":[...] },
    "data":   {"items": [...]}
  }'
POST /render-template Render a saved template by ID 
curl -X POST http://localhost:8000/render-template \
  -d '{"templateId":"invoice","data":{...},"tenant":"acme"}'
POST /render-jrxml Convert JRXML → render → PDF 
curl -X POST http://localhost:8000/render-jrxml \
  -d '{"jrxml":"examples/sample_report.jrxml","data":{...}}'
GET /health Server status + cache stats 
curl http://localhost:8000/health
# → {"status":"ok","version":"2.0.0","cache":{"entries":0,"hit_rate":0}}

🏢 SaaS

Multi-Tenant SaaS Features

Ship a whitelabel reporting engine to your customers. Each tenant gets isolated themes, templates, and parameters.

🎨

Tenant Themes

Each tenant gets a custom color palette, logo, and font. Set via themes/acme.json or the PUT /tenants/:id/theme endpoint.

📑

Template Registry

Store layouts by name. Reference as POST /render-template {"templateId":"invoice"}. Tenant-isolated — ACME can't see CorpB's templates.

⚡

Layout Cache

LRU cache with configurable TTL. Layout JSON is hashed and cached per-tenant. Rendering hot layouts takes microseconds instead of re-parsing.

# Register a template for a tenant
curl -X POST http://localhost:8000/templates \
  -d '{
    "templateId": "invoice",
    "tenant":     "acme",
    "layout":     {...layout json...}
  }'

# Render it with tenant theme applied automatically
curl -X POST http://localhost:8000/render-template \
  -d '{"templateId":"invoice","tenant":"acme","data":{...}}' \
  --output acme-invoice.pdf
